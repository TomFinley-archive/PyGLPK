GLPK Version Specific Notes

When building PyGLPK, certain functions will be disabled depending on what version of GLPK the build process believes it is using. Below is a list of GLPK versions, and descriptions of what functionality will not work in earlier versions. (Or, in some cases, later versions.)

The examples use a hypothetical glpk.LPX instance named lp for illustrative purposes.

GLPK 4.15

  • Removal of the ability to read and write GNU LP problem descriptions, glpk.LPX(glp='filename') and lp.write(glp='filename') (owing to removal of C routines lpx_read_prob and lpx_write_prob)

GLPK 4.13

  • The exact simplex solver, lp.exact() (owing to introduction of C routine lpx_exact)

GLPK 4.10

  • The Bixby basis routine, lp.cpx_basis() (owing to introduction of C routine lpx_cpx_basis)

GLPK 4.9

  • The advanced integer solver, lp.intopt() (owing to introduction of C routine lpx_intopt)
  • The integer optimality condition checker, lp.kktint() (owing to introduction of C routine lpx_check_int)

GLPK 4.7

  • Indexing rows and columns by their names (owing to introduction of C routines lpx_create_index, lpx_find_row, lpx_find_col, and lpx_delete_index), though one can always name rows and columns
  • The integer optimality condition checker, lp.kktint() (owing to introduction of C routine lpx_check_int)
  • Ability to read and write free MPS problem descriptions, glpk.LPX(freemps='filename') and lp.write(freemps='filename') (owing to introduction of C routines lpx_read_freemps and lpx_write_freemps)

GLPK 4.6

  • Ability to read and write GNU LP problem descriptions, glpk.LPX(glp='filename') and lp.write(glp='filename') (owing to introduction of C routines lpx_read_prob and lpx_write_prob)

GLPK 4.4

  • At this point there was a major reorganization of GLPK. Versions prior to GLPK 4.4 to are incompatible with PyGLPK.
Thomas Finley, 2007
